# Contributing to SafeView

감사합니다! 🙌  
SafeView에 기여하려면 다음 절차를 따라주세요.

---

## 기본 규칙
- 모든 PR은 각 레포지토리의 `dev` 브랜치를 기준으로 생성합니다.
- 커밋 메시지는 **Conventional Commits** 규칙을 따릅니다.  
  예: `feat: add access level validation`
- 문서/정책 변경 시 PR 제목에 `[docs]` 또는 `[policy]` 표시.

---

## 브랜치 네이밍
- feature/기능명 
- fix/버그명
- docs/문서명
- chore/작업명
